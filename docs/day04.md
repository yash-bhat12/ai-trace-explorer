# Day 4 - Workflow Tracing

## Objective

Introduce execution tracing into the workflow engine.

## Completed

- Created Trace and Span models
- Built a custom WorkflowTracer
- Added trace IDs to every workflow
- Recorded execution latency for each workflow step
- Displayed execution traces in the terminal
- Updated workflow response with trace information

## What I Learned

- What execution traces are
- Difference between traces and logs
- Workflow instrumentation
- Measuring latency
- Generating unique trace IDs

## Next Steps

Persist traces and expose them through an API.
